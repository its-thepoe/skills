# @its-thepoe/design-engineering

**Agent Skill** — **design engineering**: motion, easing, component feel, and UI polish in the spirit of **Emil Kowalski**–style craft (fast, intentional, restrained).

## What it does

Gives the agent a vocabulary and checklist for **micro-interactions**, transitions, hover states, popovers, toasts, drawers, and Framer Motion / CSS work—what to tune, what to avoid, and how to explain “why this feels off.” Includes **`LOVABLE.md`** for Lovable-style environments.

## Why it’s useful

- Turns vague “make it pop” into **actionable** timing, easing, and hierarchy tweaks.
- Aligns motion with **performance and accessibility** defaults (reduced motion, transform-first).
- Speeds up **review** of frontend polish without a separate design spec for every line.

## Use when

- You’re building or reviewing **UI components**, animations, or interaction details.
- You ask for **design engineering**, **motion**, **micro-interactions**, or “why does this feel cheap / sluggish / busy.”

## Install into your agents

```bash
npx @its-thepoe/skills@latest install design-engineering
```

## Contents

- `SKILL.md` — review format and principles  
- `LOVABLE.md` — condensed notes for Lovable-style workflows
