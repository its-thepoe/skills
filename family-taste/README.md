# @its-thepoe/family-taste

**Agent Skill** — **Family Values** UI philosophy: **simplicity** (gradual revelation), **fluidity** (spatial motion instead of teleports), and **delight** scaled by how often someone hits a feature.

## What it does

Frames critique and design choices around progressive disclosure, meaningful motion between states, and where “delight” earns its tax. Good for flows, sheets, empty states, live numbers, and interfaces that feel **dead** or **overwhelming**. Includes **`LOVABLE.md`** for Lovable-style workflows.

## Why it’s useful

- Gives a **coherent taste model** so polish isn’t random decoration.
- Helps decide **what to show first** vs later vs never on small screens.
- Pairs well with **design-engineering** for easing and timing specifics.

## Use when

- You’re critiquing or shaping **product UI**, onboarding, trays/sheets, empty states, or “this feels wrong but I can’t name it.”
- You mention **Family values**, **progressive disclosure**, **torph**, **liveline**, or **taste** in UI context.

## Install into your agents

```bash
npx @its-thepoe/skills@latest install family-taste
```
