# @its-thepoe/design-motion-principles

**Agent Skill** — structured **motion and interaction audits** using three lenses: **Emil Kowalski** (restraint, speed), **Jakub Krehel** (production polish), **Jhey Tompkins** (playful CSS craft)—weighted by **project context**, not one-size-fits-all rules.

## What it does

Walks reconnaissance (what the product is, what motion already exists), hunts for **motion gaps** (UI that changes state with no transition), then delivers a prioritized audit with per-designer perspectives and deep references under `references/`.

## Why it’s useful

- Surfaces **missing** motion, not only “bad” motion.
- Avoids applying productivity-tool rules to **creative** or **kids** products (and vice versa).
- Bundles **audit-checklist** + long-form references so the agent stays consistent across sessions.

## Use when

- You want a **motion audit**, interaction review, **AnimatePresence** / conditional UI pass, or per-designer motion feedback.
- You say **motion audit**, **interaction design audit**, or **design-motion-principles**.

## Install into your agents

```bash
npx @its-thepoe/skills@latest install design-motion-principles
```
